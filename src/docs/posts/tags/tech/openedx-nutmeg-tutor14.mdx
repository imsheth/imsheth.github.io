## python 3.8.10
- <a href="https://github.com/openedx/edx-platform/blob/master/Dockerfile" target="_blank" rel="noopener noreferrer">
    Picking the versions listed in the Open edX platform Dockerfile fox maximum compatibility and support
  </a>
- <a href="https://www.makeuseof.com/install-python-ubuntu/" target="_blank" rel="noopener noreferrer">
    python ppa installation
  </a>

1. Install python 3.8.10 and required packages
```shell
sudo add-apt-repository ppa:deadsnakes/ppa && sudo apt-get update && sudo apt-get -y upgrade && sudo apt-get install software-properties-common && sudo apt install build-essential zlib1g-dev libncurses5-dev libgdbm-dev libnss3-dev libssl-dev libreadline-dev libffi-dev wget libyaml-dev && sudo apt install python3.8 && sudo apt-get install python3-dev && python3.8 --version
```

2. Verify python 3.8.10 installation path
```shell
whereis python
```

<a href="/posts/openedx-nutmeg-tutor14/4.png" target="_blank" rel="noopener noreferrer">
  <img src="/posts/openedx-nutmeg-tutor14/4.png" alt="openedx-nutmeg-tutor14-4-png" align="left" style="margin-right: 2rem; margin-bottom: 1rem" />
</a>

3. Set environment/path/alias variables for python 3.8.10 & pip 22.1.2
- <a href="https://stackoverflow.com/a/60083786/3152654" target="_blank" rel="noopener noreferrer">
    Setting path variable for python 3.8.10
  </a>
- <a href="https://stackoverflow.com/a/41986556/3152654" target="_blank" rel="noopener noreferrer">
    Set alias for python to point to python 3.8.10
  </a>
- <a href="https://askubuntu.com/a/956193" target="_blank" rel="noopener noreferrer">
    Set alias for python to point to python 3.8.10 in .bashrc
  </a>
```shell
echo 'export PATH=$PATH:"/usr/bin/python3.8"' >> ~/.bashrc && echo 'alias python="python3.8"' >> ~/.bashrc && echo 'export PATH=$PATH:"/home/ubuntu/.local/bin"' >> ~/.bashrc && echo 'alias pip="pip3"' >> ~/.bashrc && source ~/.bashrc && echo $PATH && python --version
```

<a href="/posts/openedx-nutmeg-tutor14/5.png" target="_blank" rel="noopener noreferrer">
  <img src="/posts/openedx-nutmeg-tutor14/5.png" alt="openedx-nutmeg-tutor14-5-png" align="left" style="margin-right: 2rem; margin-bottom: 1rem" />
</a>

4. pip 22.1.2
- <a href="https://pip.pypa.io/en/stable/installing/" target="_blank" rel="noopener noreferrer">
    pip installation
  </a>
- <a href="https://stackoverflow.com/a/61717037/3152654" target="_blank" rel="noopener noreferrer">
    pip installation for python 3.8
  </a>

```shell
cd /tmp && curl https://bootstrap.pypa.io/get-pip.py -o get-pip.py && python get-pip.py && pip install --upgrade pip --user && pip --version
```

## Docker 20.10.17
- <a href="https://docs.tutor.overhang.io/install.html#requirements" target="_blank" rel="noopener noreferrer">
  Install docker (v18.06.0+) and docker compose (v1.22.0+)
  </a>
- <a href="https://docs.docker.com/engine/install/ubuntu/" target="_blank" rel="noopener noreferrer">
  Docker installation
  </a>
- <a href="https://www.digitalocean.com/community/tutorials/how-to-install-and-use-docker-compose-on-ubuntu-20-04" target="_blank" rel="noopener noreferrer">
  Docker compose installation
  </a>
- <a href="https://github.com/docker/compose/releases/" target="_blank" rel="noopener noreferrer">
  Docker compose releases
  </a>
- <a href="https://askubuntu.com/a/739861/418976" target="_blank" rel="noopener noreferrer">
  Run docker without sudo
  </a>

1. Install docker 20.10.17
```shell
cd /tmp && curl -fsSL https://get.docker.com -o get-docker.sh && sh get-docker.sh && docker --version
```
<a href="/posts/openedx-nutmeg-tutor14/7.png" target="_blank" rel="noopener noreferrer">
  <img src="/posts/openedx-nutmeg-tutor14/7.png" alt="openedx-nutmeg-tutor14-7-png" align="left" style="margin-right: 2rem; margin-bottom: 1rem" />
</a>

2. <a href="https://blog.usejournal.com/how-to-enable-docker-remote-api-on-docker-host-7b73bd3278c6" target="_blank" rel="noopener noreferrer">
  Enable docker remote API on docker host (comments section)
  </a>
```shell
sudo vim /lib/systemd/system/docker.service
```

```shell
# Comment the original line for backup to revert if needed
#ExecStart=/usr/bin/dockerd -H fd:// --containerd=/run/containerd/containerd.sock
ExecStart=/usr/bin/dockerd -H fd:// -H 0.0.0.0:2375 --containerd=/run/containerd/containerd.sock
```

3. Reload - restart daemon and verify docker remote API response
```shell
sudo systemctl daemon-reload
sudo service docker restart
curl http://localhost:2375/images/json
```
<a href="/posts/openedx-nutmeg-tutor14/8.png" target="_blank" rel="noopener noreferrer">
  <img src="/posts/openedx-nutmeg-tutor14/8.png" alt="openedx-nutmeg-tutor14-8-png" align="left" style="margin-right: 2rem; margin-bottom: 1rem" />
</a>

4. Install docker compose 1.29.2
```shell
sudo curl -L "https://github.com/docker/compose/releases/download/1.29.2/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose && sudo chmod +x /usr/local/bin/docker-compose && docker-compose --version
```

<a href="/posts/openedx-nutmeg-tutor14/9.png" target="_blank" rel="noopener noreferrer">
  <img src="/posts/openedx-nutmeg-tutor14/9.png" alt="openedx-nutmeg-tutor14-9-png" align="left" style="margin-right: 2rem; margin-bottom: 1rem" />
</a>

5. Run docker without sudo
```shell
sudo groupadd docker && sudo usermod -aG docker ${USER}
```

<a href="/posts/openedx-nutmeg-tutor14/10.png" target="_blank" rel="noopener noreferrer">
  <img src="/posts/openedx-nutmeg-tutor14/10.png" alt="openedx-nutmeg-tutor14-10-png" align="left" style="margin-right: 2rem; margin-bottom: 1rem" />
</a>

6. This completes docker 20.10.17 and docker compose 1.29.2 installation and setup

1. Turning off rate limiting
```shell
tutor config save --set RATELIMIT_ENABLE=False
```

2. Turning on debugging
```shell
tutor config save --set DEBUG=True
```

3. Using Google SMTP
```shell
tutor config save --set RUN_SMTP=false

tutor config save \
 --set SMTP_HOST=smtp.gmail.com \
 --set SMTP_PORT=587 \
 --set SMTP_USE_SSL=false \
 --set SMTP_USE_TLS=true \
 --set SMTP_USERNAME=gmailid@gmail.com \
 --set SMTP_PASSWORD=gmailidpassword
```

4. Applying settings
```shell
tutor local restart
```
